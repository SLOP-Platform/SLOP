# SLOP v3 — Docker Installation

## What this installs

SLOP v3 is a **self-hosted media stack manager**. It runs as a single Docker container that manages all your other containers (Sonarr, Radarr, Plex, Jellyfin, Prowlarr, etc.) through the Docker socket.

```
                  ┌─────────────────────────┐
                  │  SLOP v3 container │
                  │  ─ Web UI  (port 8080)   │
                  │  ─ REST API              │
Browser ─────────►│  ─ Health scheduler      │──► Docker socket
                  │  ─ CF Tunnel integration │      │
                  └─────────────────────────┘      │
                                                    ▼
                  ┌──────────┐  ┌──────────┐  ┌──────────┐
                  │  Traefik │  │  Sonarr  │  │  Radarr  │  ...
                  │ (proxy)  │  │          │  │          │
                  └──────────┘  └──────────┘  └──────────┘
```

## Prerequisites

| Requirement | Minimum | Notes |
|-------------|---------|-------|
| OS | Ubuntu 22.04 / Debian 12 | Any Linux with Docker works |
| Docker Engine | 24.0+ | **Not** Docker Desktop |
| Docker Compose | v2 (built-in) | `docker compose` not `docker-compose` |
| RAM | 2 GB | 4 GB recommended when using LLM health agent |
| Disk | 20 GB free | For config; media lives on your NAS/drives |
| Domain name | Required | For HTTPS — Cloudflare DNS recommended |

### Install Docker (Ubuntu)

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Log out and back in for group to take effect
docker --version   # should print 24+
```

## Quick start (5 steps)

### Step 1 — Clone the repository

```bash
git clone https://github.com/Nnyan/SLOP.git
cd slop
```

### Step 2 — Create directories

These paths must be identical inside and outside the manager container so the
compose files it generates point to real host paths.

```bash
# Config storage — all app config folders live here
sudo mkdir -p /srv/slop/config

# Media library — your movies, TV, music, books
# Skip if using a NAS mount (configure that separately)
sudo mkdir -p /mnt/media

# Give your user write access
sudo chown -R $USER:$USER /srv/slop/config
```

### Step 3 — Configure environment

```bash
cp .env.example .env
```

Open `.env` and set at minimum:

```bash
# Your actual paths (must match what you created above)
CONFIG_ROOT=/srv/slop/config
MEDIA_ROOT=/mnt/media

# Your timezone — see: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones
TZ=America/Los_Angeles

# Docker group ID (for socket access)
DOCKER_GID=$(stat -c '%g' /var/run/docker.sock)
echo "DOCKER_GID=$DOCKER_GID" >> .env   # adds it automatically

# Strong Postgres password (required if you'll install Umami, Komodo, etc.)
POSTGRES_PASSWORD=$(openssl rand -base64 24)
echo "POSTGRES_PASSWORD=$POSTGRES_PASSWORD" >> .env
```

For HTTPS certificates, add your Cloudflare API token (see [Getting Cloudflare credentials](#getting-cloudflare-credentials)):

```bash
CF_DNS_API_TOKEN=your_token_here
CF_ACCOUNT_ID=your_account_id
CF_TUNNEL_ID=your_tunnel_id   # only if using CF Tunnel
CF_ZONE_ID=your_zone_id
```

### Step 4 — Start SLOP

```bash
docker compose up -d
docker compose logs -f   # watch startup logs, Ctrl+C to stop tailing
```

Wait for:
```
INFO  backend.api.main  SLOP backend ready — db: /data/state.db
```

### Step 5 — Open the UI and run the wizard

Open **http://your-server-ip:8080** in your browser.

Click **Setup** in the sidebar and fill in:
- Your domain (e.g. `example.com`)
- Config root: `/srv/slop/config`
- Media root: `/mnt/media`
- DNS provider: `cloudflare` (or your provider)
- ACME email: your email address

Click **Run setup wizard**. Traefik starts, the wildcard certificate is issued, and the platform is ready.

## What to do after setup

```bash
# Check everything started
docker compose ps
curl http://localhost:8080/api/ping   # should return {"status":"ok"}

# Use the CLI
./ms status            # platform overview
./ms apps list         # installed apps (empty to start)
./ms catalog           # browse available apps

# Install your first apps
./ms apps install sonarr
./ms apps install radarr
./ms apps install prowlarr

# Or use the web UI — Catalog tab → Install
```

## Deploy infra slots (recommended)

In the UI → **Infrastructure**, deploy:

1. **Auth** → TinyAuth — adds login to your apps
2. **Tunnel** → Cloudflare Tunnel — secure external access, no port forwarding
3. **Management** → Dockhand or Portainer — container management UI

## Getting Cloudflare credentials

The wizard needs these to issue wildcard HTTPS certificates via DNS-01 challenge.

**CF_DNS_API_TOKEN** (required for HTTPS):
1. Log into [Cloudflare dashboard](https://dash.cloudflare.com) → My Profile → API Tokens
2. Click **Create Token** → Use template **Edit zone DNS**
3. Permissions: `Zone / DNS / Edit`, `Zone / Zone / Read`
4. Zone Resources: Include → Specific zone → your domain
5. Copy the generated token → paste into `.env`

**CF_ZONE_ID** (required for HTTPS):
1. Cloudflare dashboard → your domain → Overview tab
2. Right sidebar → Zone ID → copy it

**CF_ACCOUNT_ID** (required if using CF Tunnel):
1. Cloudflare dashboard → any page → right sidebar → Account ID → copy it

**CF_TUNNEL_ID** (required if using CF Tunnel):
1. Cloudflare Zero Trust → Networks → Tunnels → Create a tunnel
2. Give it a name (e.g. `slop`)
3. Copy the tunnel ID from the URL — looks like a UUID
4. Also copy the **tunnel token** — you'll paste this in the UI when deploying the Tunnel infra slot

## Configuration reference

| Variable | Default | Description |
|----------|---------|-------------|
| `CONFIG_ROOT` | `/srv/slop/config` | Where app config folders are created |
| `MEDIA_ROOT` | `/mnt/media` | Root of your media library |
| `MS_PORT` | `8080` | Host port for the web UI and API |
| `TZ` | `UTC` | Timezone for all containers |
| `DOCKER_GID` | `999` | GID of the docker group on your host |
| `CF_DNS_API_TOKEN` | — | Cloudflare API token for DNS-01 cert challenge |
| `CF_ZONE_ID` | — | Cloudflare zone ID for your domain |
| `CF_ACCOUNT_ID` | — | Cloudflare account ID (for tunnel) |
| `CF_TUNNEL_ID` | — | Cloudflare tunnel UUID |
| `POSTGRES_PASSWORD` | — | Password for the shared Postgres service |
| `PLEX_CLAIM` | — | Plex claim token (get at plex.tv/claim, expires in 4 min) |

## Troubleshooting

**Can't connect to Docker socket:**
```bash
# Check your DOCKER_GID is correct
stat -c '%g' /var/run/docker.sock
# Update .env and restart
docker compose up -d
```

**Port 8080 already in use:**
```bash
# Change to port 8081
echo "MS_PORT=8081" >> .env
docker compose up -d
# Access at http://your-server:8081
```

**Certificate not issuing:**
- Verify `CF_DNS_API_TOKEN` has Zone / DNS / Edit permission
- Wizard must complete fully before certs are requested
- Let's Encrypt rate limits: 5 certs per domain per week on staging, more on prod
- Check logs: `docker compose logs slop | grep -i acme`

**Apps not appearing after install:**
```bash
docker compose logs slop | tail -50
./ms health status
```

**Reset and start over:**
```bash
docker compose down
docker volume rm slop_slop_data
docker compose up -d
# Then re-run the wizard
```

## Non-Cloudflare DNS providers

Replace `cloudflare` in the wizard with your provider key. Common examples:

| Provider | Wizard key | Required env vars |
|----------|-----------|-------------------|
| AWS Route 53 | `route53` | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION` |
| Namecheap | `namecheap` | `NAMECHEAP_API_USER`, `NAMECHEAP_API_KEY` |
| Porkbun | `porkbun` | `PORKBUN_API_KEY`, `PORKBUN_SECRET_API_KEY` |
| DigitalOcean | `digitalocean` | `DO_AUTH_TOKEN` |
| DuckDNS | `duckdns` | `DUCKDNS_TOKEN` |

Full provider list: `GET /api/platform/dns-providers` or the wizard dropdown.

## CGNAT / no port forwarding (Starlink, etc.)

If your ISP uses CGNAT, you can't open port 443 for direct media server access.

**Option A — Tailscale** (recommended):
1. Install Tailscale infra slot in the UI → Infrastructure → Tunnel → Deploy
2. All apps accessible via `https://app.yourtailnet.ts.net`
3. No port forwarding needed

**Option B — Cloudflare Tunnel only** (web UIs only, not media streaming):
- Plex, Jellyfin, and Emby cannot route through CF Tunnel (ToS violation for video)
- Use Tailscale for media, CF Tunnel for management UIs

## Migrating from SLOP-RAD v2

If you have v2 running at `/opt/msrad`:

```bash
# Preview what would be migrated (no changes made)
python3 tools/migrate_from_v2.py --dry-run

# Run the migration (v2 keeps running)
pip install pyyaml   # required to read v2 docker-compose.yml
python3 tools/migrate_from_v2.py

# Verify
./ms apps list
./ms health status
```
