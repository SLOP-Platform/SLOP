## Update model

The SLOP install directory (`/opt/slop`) is an **HTTPS git clone** of
`Nnyan/SLOP`, owned by the **service user `slop`**. Updates pull from
`origin/main` using `ms-update` or `deploy.sh --update`; there is no manual
`scp`/`rsync` copy step.

```bash
# Standard update (recommended)
sudo ms-update

# Alternative: use the deploy script
sudo -u slop /opt/slop/deploy.sh --update
```

Both tools:
1. Run all file-touching ops (`git fetch`, `pip install`, `npm ci && npm run build`)
   as the service user (`sudo -u slop ...`).
2. Perform a `fetch` + fast-forward attempt; fall back to `reset --hard origin/main`
   automatically if the clone is diverged.
3. Rebuild the frontend (`backend/static/` is gitignored and deleted by `reset --hard`;
   the build always runs when frontend files changed).
4. Restart the service with `systemctl restart slop` (root-only op).

For the full ownership model, shared helper functions (`tools/deploy_lib.sh`), and a
manual recovery runbook for diverged clones, see [DEPLOY.md](DEPLOY.md).

## Service port

The canonical service-port environment variable is **`MS_PORT`** (default `8080`).

`deploy.sh` writes `MS_PORT=<value>` into `/opt/slop/.env` and bakes the port
into the generated systemd unit. To change the port:

```bash
# Edit .env
sed -i 's/^MS_PORT=.*/MS_PORT=9090/' /opt/slop/.env
# Regenerate the unit with the new port (or edit /etc/systemd/system/slop.service directly)
systemctl restart slop
```

The service port is set via `MS_PORT` (default 8080).

## Docker socket and root-equivalent host privileges

SLOP mounts `/var/run/docker.sock` so the platform wizard can manage Traefik
and installed app containers.  Access to the Docker socket is
**root-equivalent on the host**: any process that reaches it can start
privileged containers, bind-mount arbitrary host paths, and escape the
container namespace entirely.

Keep this in mind when:

- Deciding which users belong to the `docker` group on the host.
- Evaluating whether to run SLOP behind an auth layer (recommended — enable
  an auth provider in the wizard's infra slot).
- Considering network exposure of the SLOP management port (`MS_PORT`).

An optional Docker socket proxy (`step_socket_proxy` in the platform wizard,
using `tecnativa/docker-socket-proxy`) can limit Traefik's API surface to
read-only operations.  That hardening step is not wired into the automated
setup yet; see the function docstring in `backend/platform/wizard.py` for
details on enabling it manually.

## Optional feature flags

| Variable | Default | Effect |
|---|---|---|
| `MS_DISABLE_DOCS` | `false` | Set to `true` to disable the Swagger UI (`/api/docs`). Requests to `/api/docs` return 404. |
| `MS_DISABLE_METRICS` | `false` | Set to `true` to disable the Prometheus `/metrics` endpoint. Useful for deployments that do not run a Prometheus scraper. |

Example:

```bash
echo 'MS_DISABLE_DOCS=true' >> /opt/slop/.env
systemctl restart slop
```

## Operator environment

<!-- OPERATOR-ENV: .env is authoritative -->
Operator env `MS_TRUSTED_HOSTS` / `DOMAIN` is authoritative in the install-dir `.env`
(`/opt/slop/.env`); the canonical edit point is: edit `.env` then
`systemctl restart slop`.

The systemd unit generated by `deploy.sh` carries `EnvironmentFile=/opt/slop/.env`,
which injects every `KEY=VALUE` into the process environment. `MS_TRUSTED_HOSTS` and
`DOMAIN` are read via `os.environ` in the FastAPI startup code, so values in `.env`
take effect on the next `systemctl restart slop`.

```bash
# Example: trust a LAN hostname
echo 'MS_TRUSTED_HOSTS=myserver.local,192.168.1.50' >> /opt/slop/.env
systemctl restart slop
```
